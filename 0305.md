在 Linux 服务器上，查看 Milvus 端口占用和映射情况有以下几种最直接的方法：

### 1. 通过 Docker 命令查看（最推荐）
由于你的 Milvus 是通过 Docker 部署的，直接查看容器映射关系是最准确的：
```bash
docker ps --format "table {{.Names}}\t{{.Ports}}"
```
**输出示例：**
*   `milvus-standalone`: `0.0.0.0:19530->19530/tcp, 0.0.0.0:9091->9091/tcp`
*   `milvus-minio`: `0.0.0.0:9000->9000/tcp, 0.0.0.0:9001->9001/tcp`
*   `milvus-etcd`: `2379/tcp, 2380/tcp`

---

### 2. 查看宿主机端口监听状态
如果你想确认端口是否已经在 Linux 宿主机上开启：

**方法 A：使用 `netstat` (常用)**
```bash
netstat -tunlp | grep -E '19530|9000|9091'
```

**方法 B：使用 `ss` (更现代，速度快)**
```bash
ss -tunlp | grep -E '19530|9000|9091'
```
*   `19530`: Milvus gRPC 服务（Spring Boot 连接这个端口）。
*   `9091`: Milvus 管理/健康检查接口。
*   `9000`: Minio API 端口。

---

### 3. 查看特定端口被哪个进程占用
如果你怀疑端口冲突（比如之前的 9000 端口），可以使用：
```bash
lsof -i:19530
lsof -i:9000
```

---

### 4. 核心端口清单 (Milvus RAG 系统)

在你的 4G 内存服务器上，请确保以下端口状态正常：

| 端口号 | 组件 | 用途 | 必须放行？ |
| :--- | :--- | :--- | :--- |
| **19530** | milvus-standalone | **核心接口**，Java 代码通过它读写向量 | **是** (内网/127.0.0.1) |
| **9091** | milvus-standalone | 健康检查、监控指标 | 可选 |
| **9000** | milvus-minio | Minio 存储数据接口 | 容器间必须互通 |
| **9001** | milvus-minio | Minio 管理后台界面 | 可选 |
| **2379** | milvus-etcd | Etcd 元数据存储 | 仅容器内互通 |

---

### 💡 针对你刚才 `Connection Timeout` 报错的排查建议

如果你在宿主机上运行 `netstat -tunlp | grep 19530` 看到结果是：
`tcp6  0  0 :::19530  :::*  LISTEN`

**这意味着：**
1.  **端口已开启：** Milvus 没问题。
2.  **网络配置问题：** 如果 Java 连不上，请再次确认 `application.yml` 里的 `host` 是 `127.0.0.1` 还是公网 IP。
3.  **内网互通：** 如果你在服务器上跑 `curl localhost:19530`（虽然它返回乱码），只要没报 `Connection Refused`，就说明端口是通的。

**操作命令总结：**
```bash
# 1. 检查容器运行
docker ps

# 2. 检查端口监听
sudo ss -tlpn | grep 19530

# 3. 检查防火墙是否允许
sudo ufw status # 或者检查阿里云安全组
```
